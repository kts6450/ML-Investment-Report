## 슬라이드 3: 왜 강화학습인가?

| 접근법 | 방식 | 한계 |
|--------|------|------|
| 지도학습 | 가격 예측 (분류/회귀) | 예측 ≠ 투자 결정 |
| 전통 최적화 | 마코위츠 등 | 고정 가정, 적응 불가 |
| **강화학습** | 매일 비중을 직접 결정 | 시행착오로 최적 전략 학습 |

**핵심**: 가격을 예측하지 않고, "이런 시장 상태에서는 어떤 비중이 최적인가"를 경험으로 학습

## 슬라이드 4: RL 환경 설계 – 전체 구조

```
Agent (PPO)
   │
   ├─ 관찰(State): 기술지표 22개 × 20일 + 현재 비중 3개 = 443차원
   │
   ├─ 행동(Action): 3개 실수 → softmax → BTC/ETH/CASH 비중 (합=1)
   │
   └─ 보상(Reward): 일간 수익률 - 거래비용
```

→ Gymnasium 기반 커스텀 환경 (`PortfolioEnv`) 구현

## 슬라이드 5: State 설계

**관측 벡터 (443차원)**

| 구성 | 차원 | 설명 |
|------|------|------|
| 기술지표 | 22 × 20 = 440 | 과거 20일간 정규화된 피처 |
| 현재 비중 | 3 | BTC, ETH, CASH 보유 비중 |

- 현재 비중을 State에 포함시켜 **리밸런싱 비용을 고려한 의사결정** 가능
- Window 방식으로 시계열 패턴 포착

---

## 슬라이드 6: Action 설계

**연속 행동 공간 (3차원)**

```
raw action: [-0.5, 0.8, 0.2]  (PPO 출력)
     │
     ▼ softmax
weights: [0.15, 0.55, 0.30]  (BTC 15%, ETH 55%, CASH 30%)
```

- softmax로 합이 항상 1이 되도록 보장
- 에이전트가 자유롭게 비중 조절 가능

---

## 슬라이드 7: Reward 설계 (1차)

**1차 버전: 단순 수익률**

```
reward = 포트폴리오 일간 수익률 - 거래비용
```

- 거래비용: 비중 변화량 × 0.1% (실제 거래소 수수료 반영)
- 단순하지만 학습 신호가 약한 문제 발견

---

## 슬라이드 8: PPO 알고리즘 선택 이유

**왜 PPO인가?**

| 알고리즘 | 장점 | 단점 |
|----------|------|------|
| DQN | 안정적 | 이산 행동만 가능 |
| A2C | 빠름 | 불안정 |
| SAC | 연속 행동 지원 | 복잡 |
| **PPO** | 연속 행동 + 안정적 + 구현 쉬움 | - |

- **연속 행동 공간** (비중은 연속값) 지원
- Clipping으로 **안정적 학습** 보장
- Stable-Baselines3에서 검증된 구현 제공

## 슬라이드 10: 1차 백테스트 결과

| 지표 | PPO (1차) | BTC Buy-and-Hold |
|------|:---------:|:-----------------:|
| 누적 수익률 | -47.68% | -39.57% |
| Sharpe Ratio | -2.1728 | -1.3354 |
| Max Drawdown | -51.83% | -49.53% |

**문제점 발견:**
1. PPO가 BTC 단순 보유보다 **더 나쁜 성과**
2. 비중 전환이 너무 빈번 → 거래비용 과다
3. Reward 신호가 너무 약해 학습이 불충분

---

## 슬라이드 11: 문제 분석

**차트 관찰 (backtest_result.png 첨부)**

1. **과도한 리밸런싱**: 매일 비중이 급격히 변동 → 거래비용 누적
2. **학습 부족**: 100k timesteps로는 443차원 관측 공간을 충분히 탐색 못 함
3. **Reward 스케일**: 일간 수익률은 ±0.01~0.05 수준으로 학습 신호가 미미

## 슬라이드 12: 3개월차 개선 계획

| 개선 항목 | 방법 |
|-----------|------|
| Reward 재설계 | Differential Sharpe Ratio 기반 보상 |
| 거래비용 페널티 강화 | turnover × 수수료 × 3배 |
| 네트워크 확대 | 256-256-128 (3층) |
| 학습량 증가 | 100k → 500k timesteps |
| 학습률 스케줄링 | Linear decay (3e-4 → 0) |
| PPO 튜닝 | entropy coef, clip range 등 |

### 발표 예상 질문 & 답변

**Q: 하락장에서 성과가 나쁜 건 당연한 거 아닌가?**
> 맞습니다. 하지만 CASH 자산이 있으므로 하락장에서 CASH 비중을 높이면 손실을 줄일 수 있습니다. 현재 1차 모델은 이를 충분히 학습하지 못했고, 3개월차에서 개선할 예정입니다.

**Q: 왜 softmax를 쓰나?**
> 포트폴리오 비중은 0 이상이고 합이 1이어야 합니다. softmax는 이 두 조건을 자연스럽게 만족시킵니다.

**Q: train/test를 시간순으로 분할하는 이유는?**
> 미래 데이터가 학습에 유출(data leakage)되는 것을 방지합니다. 과거 80%로 학습하고 미래 20%로 평가해야 실전 성능을 정확히 측정할 수 있습니다.
