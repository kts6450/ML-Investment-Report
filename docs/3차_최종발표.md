## 슬라이드 4: 개선 1 – Reward 재설계

**기존**: 단순 일간 수익률
```
reward = portfolio_return - tx_cost
```

**개선**: Differential Sharpe Ratio + 스케일업 + 강화된 페널티
```
reward = diff_sharpe × 0.5 + port_return × 10.0 - turnover_penalty
```

| 요소 | 역할 |
|------|------|
| Differential Sharpe | 리스크 대비 수익률 (온라인 EMA 방식) |
| port_return × 10.0 | 수익률 신호 증폭 |
| turnover_penalty (×3) | 불필요한 매매 억제 |

## 슬라이드 6: 개선 3 – 네트워크 구조

**기존**: 기본 MLP (64-64, ReLU)

**개선**: Custom MLP (256-256-128, Tanh)

| 항목 | 기존 | 개선 |
|------|------|------|
| 레이어 | 2층 (64-64) | 3층 (256-256-128) |
| 활성함수 | ReLU | Tanh |
| 파라미터 수 | ~8k | ~130k |

- Tanh: 출력 범위가 [-1, 1]로 제한되어 포트폴리오 비중 문제에 적합
- 더 깊은 네트워크로 443차원 관측 공간의 패턴을 충분히 학습

## 슬라이드 7: 개선 4~6 – 학습 설정 튜닝

| 파라미터 | 기존 | 개선 |
|----------|------|------|
| Total Timesteps | 100,000 | **500,000** |
| Learning Rate | 3e-4 (고정) | **3e-4 → 0 (linear decay)** |
| n_steps | 256 | **512** |
| batch_size | 64 | **128** |
| n_epochs | 10 | **15** |
| gamma | 0.99 | **0.995** |
| gae_lambda | 0.95 | **0.98** |
| clip_range | 0.2 (고정) | **0.2 → 0 (linear decay)** |
| ent_coef | 0.0 | **0.005** |

- **Linear decay**: 학습 초반에는 크게 탐색, 후반에는 미세 조정
- **Entropy coef**: 적절한 탐색-활용 균형 유지
- **높은 gamma**: 장기 수익률을 더 중시

### 발표 예상 질문 & 답변

**Q: 하락장에서 이긴 건 의미가 있는가?**
> 하락장에서 손실을 줄이는 것이 포트폴리오 관리의 핵심입니다. "돈을 더 버는 것"뿐 아니라 "덜 잃는 것"도 알파(alpha)입니다. 특히 MDD가 5%p 줄어든 것은 투자자의 심리적 안정과 복리 효과 측면에서 큰 의미가 있습니다.

**Q: Differential Sharpe Ratio란?**
> 전통적 Sharpe Ratio는 전체 기간의 평균/표준편차를 사용하지만, RL에서는 매 스텝마다 보상이 필요합니다. EMA(지수이동평균)로 수익률의 평균과 분산을 온라인으로 추적하여, 매 스텝의 Sharpe 변화량을 근사한 것입니다.

**Q: 500k timesteps가 충분한가?**
> 한 에피소드가 약 880스텝이므로, 500k는 약 568에피소드에 해당합니다. EvalCallback으로 학습 중 성능이 수렴하는 것을 확인했습니다. 더 늘려도 큰 개선은 없을 것으로 판단됩니다.

**Q: 실거래에 적용할 수 있는가?**
> 현재 시스템은 슬리피지, 유동성, 주문 체결 지연 등을 반영하지 않으므로 바로 실거래에 적용하기는 어렵습니다. 하지만 추천 비중을 참고 지표로 활용하는 것은 가능합니다.

**Q: 왜 PPO가 DQN보다 적합한가?**
> DQN은 이산 행동만 지원하므로 "BTC 10%, 20%, 30%..." 같이 비중을 이산화해야 합니다. 3개 자산 × 10단계만 해도 1,000개 조합이 됩니다. PPO는 연속 행동 공간을 직접 다루므로 포트폴리오 비중 문제에 훨씬 자연스럽습니다.
