# 다른 컴퓨터에서 Cursor에 붙여넣을 프롬프트

아래 내용을 그대로 복사해서 Cursor 채팅에 붙여넣으면 됩니다.

---

## 프롬프트 시작

```
기존에 진행한 강화학습 기반 가상자산 포트폴리오 최적화 프로젝트가 있어.
GitHub에 코드가 올라가 있고, 이걸 기반으로 작업을 이어가려고 해.

### 저장소
https://github.com/kts6450/ML-Investment-Report.git

### 프로젝트 개요
- BTC, ETH, CASH 3개 자산에 대해 PPO 강화학습으로 최적 포트폴리오 비중을 결정하는 시스템
- 가격 예측이 아니라 자산 비중 allocation 문제로 설계
- Binance API로 데이터 수집 → Feature Engineering → PPO 학습 → 백테스트 → FastAPI → n8n 알림 → Streamlit 대시보드

### 현재 완성된 구조
ML/
├── crypto_portfolio/          # 메인 패키지
│   ├── config/settings.py     # 전역 설정
│   ├── config/database.py     # SQLAlchemy
│   ├── data/collector.py      # Binance/Upbit API → CSV/DB 저장
│   ├── data/loader.py         # CSV/DB → DataFrame 로드
│   ├── data/models.py         # DB ORM 모델
│   ├── features/engineering.py # 22개 기술지표 + Z-score 정규화
│   ├── envs/portfolio_env.py  # Gymnasium 환경 (Diff. Sharpe Reward)
│   ├── training/train.py      # PPO 학습 (256-256-128, 500k steps)
│   ├── backtest/backtester.py # 백테스트 + 시각화
│   └── api/server.py          # FastAPI 추론 서버
├── dashboard.py               # Streamlit 대시보드
├── run_pipeline.py            # 파이프라인 실행 스크립트
├── n8n_workflow_example.json  # n8n 워크플로
├── docs/                      # 발표 자료 (1차/2차/3차)
├── requirements.txt
└── .env.example

### 강화학습 설계
- State: 기술지표 22개 × 20일 + 현재 비중 3개 = 443차원
- Action: 3개 실수 → softmax → BTC/ETH/CASH 비중 (합=1)
- Reward: diff_sharpe × 0.5 + port_return × 10.0 - turnover_penalty (×3)
- Network: MLP 256-256-128 (Tanh)
- 학습: 500k timesteps, linear decay LR, gamma=0.995

### 백테스트 결과
- PPO 누적 수익률: -31.55% (BTC B&H: -37.93%) → PPO가 6.4%p 우위
- PPO Sharpe: -1.20 (BTC B&H: -1.27) → PPO 우위
- PPO MDD: -44.67% (BTC B&H: -49.53%) → PPO 4.9%p 개선

### 기술 스택
- Python 3.14, Gymnasium, Stable-Baselines3 (PPO), PyTorch
- Pandas, NumPy, FastAPI, Streamlit, Plotly, Matplotlib
- Binance API, n8n, Discord Webhook

### 실행 방법
1. git clone 후 pip install -r requirements.txt
2. cp .env.example .env
3. py run_pipeline.py --step collect  (데이터 수집)
4. py run_pipeline.py --step train   (PPO 학습)
5. py run_pipeline.py --step backtest (백테스트)
6. py run_pipeline.py --step serve   (API 서버)
7. py -m streamlit run dashboard.py  (대시보드)

이 프로젝트를 기반으로 [여기에 하고 싶은 작업 설명] 해줘.
```

---

## 사용 방법

1. 다른 컴퓨터에서 Cursor 열기
2. 위 프롬프트를 복사
3. `[여기에 하고 싶은 작업 설명]` 부분을 원하는 내용으로 바꾸기
4. Cursor 채팅에 붙여넣기
